# Changelog

All notable changes to this project. The repo is tagged at the `v2`–`v4`
milestones below.

## Unreleased

### Added
- Project showcase docs: `LICENSE` (MIT), `AGENTS.md` / `CLAUDE.md` agent
  guides, this changelog, README badges/logo, and crate metadata.

## v4 — 2026-06-11

### Added
- macOS `.app` packaging via `package-macos.sh`, with `--universal` and `--dmg`
  options.
- App icon (an eye): embedded in the macOS `.app` (`.icns`) and the Windows
  `.exe` (`build.rs` + `winresource`); assets generated by `tools/gen_icon.py`.
- Cross-platform CI (`.github/workflows/build.yml`) building native macOS,
  Linux and Windows executables.

### Changed
- `winit` is now a macOS-only dependency (used only for the no-Dock-icon policy).

## v3 — 2026-06-11

### Added
- Per-reminder **duration unit** (seconds or minutes), mirroring the interval.
- Optional **click-to-dismiss**, configurable **per reminder**: while such a
  reminder is shown the overlay becomes clickable and a click dismisses it;
  otherwise it stays fully click-through.

### Changed
- Each reminder in the Settings window gets its own full-width message field.

## v2 — 2026-06-11

### Added
- In-app **Settings window** (opened from the tray) to add/remove reminders and
  edit messages, intervals, durations and appearance, saved back to the config.
- Interval **unit** selector (seconds / minutes / hours).

### Fixed
- Tray menu items did nothing: `muda` delivers events to a handler *or* the
  channel, never both — actions are now resolved inside the handler.

## v1 — 2026-06-11

### Added
- Initial release: transparent, click-through, always-on-top overlay that fades
  semi-transparent reminders in at random spots; multiple reminders driven by a
  TOML config; system-tray Pause/Resume and Quit; macOS accessory (no Dock icon).
